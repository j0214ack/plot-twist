# PlotTwist

> 把一段卡住人的生活故事，變成可以親手走過的另一個視角。

PlotTwist 是團隊目前的工作名稱。選題尚未定案，其中一個主要候選是 AI-native 互動敘事實驗：使用者提供一段生活事件，AI 協助拆開「發生了什麼」與「我如何解讀」，再把其中的張力、角色、象徵與另一種觀看方式，編譯成一段短小、可操作、可重玩的網頁體驗。

若選擇敘事方向，要驗證的不是「AI 能不能自動做很多遊戲」，而是：**互動本身能不能讓人感受到，一件事不只存在一種故事。**

## 目前狀態

`Ideation / Hackathon POC`

目前有三個候選專案主題：

- 個人故事 → 視角轉換 → 可重玩的互動敘事。
- 讓一般人量產網頁小遊戲的互動藝術創作工具。
- 以 OpenAP 為基礎，提供 public URL 與開發資訊的前端 DevTool。

這三者是選題階段的獨立候選，不是一套必須融合的產品架構。敘事體驗可以借用遊戲模板技術；OpenAP DevTool 則是另一個專案選項，不是 PlotTwist 的底層或 roadmap。

完整的討論脈絡、候選方向與 POC 建議請見 [概念文件](docs/concept.md)。

## Hackathon

本專案預計參加 [OpenAI Build Week Community Hackathon - Taipei](https://luma.com/build-week-taipei)。活動重視可運作的 demo、清楚的使用者 workflow、對 Codex／OpenAI tools 的運用、真實使用者價值，以及負責任且可持續的 AI 設計。

## 原則

- 讓使用者保有對自己故事的解釋權；AI 提供可能性，不宣告真相。
- 先做有明確體驗目的的作品，再擴充生成種類。
- 以可控模板保住美術方向、互動品質、速度與 demo 穩定性。
- 未經明確同意，不公開或長期保存使用者輸入的私人故事。
- 在沒有臨床合作與驗證前，不把產品稱為心理治療或 digital therapeutic。
