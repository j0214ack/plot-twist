# Team PoCs

團隊在下一次收斂前，各自製作一個可以獨立運作的 PoC。這個資料夾用來保存每個實驗的假設、設計、實作與驗證結果，而不是提前把其中任一方向定為 PlotTwist 的正式產品架構。

## 目前的 PoC

| PoC | 核心問題 | 狀態 |
| --- | --- | --- |
| [The Unwritten Spell／未寫之咒](the-unwritten-spell/README.md) | 玩家能否用自然語言，讓 AI 現場寫出原本不存在、可以立即遊玩的遊戲機制？ | 規格草案 |

## 建議的資料夾內容

每個 PoC 可以依需要包含：

- `README.md`：一句話、假設與目前狀態。
- 設計文件：玩家體驗、世界規則與重要取捨。
- 技術文件：原型架構、介面與限制。
- 驗證計畫：測試情境、成功標準與實驗結果。
- 原型程式：能獨立啟動與重現的 demo。

PoC 應優先留下「驗證到什麼」與「什麼失敗了」，不要只保存最後一段漂亮影片。
